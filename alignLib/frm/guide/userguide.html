FRM User Guide
There are 3 tools in the FRM set: frmr, frmv, and frm6. The first two perform a rotational matching between a density map and an atomic structure (single molecule docking), whereas the third does a full rigid body docking (fitting of an atomic structure inside an EM map).
frmr  and  frmv -  FFT-accelerated 3D exhaustive rotational search

Purpose:

These utilities offer robust, accurate and fast rigid-body rotational search tools. The first one, frmr, radializes the maps to be fitted, and thus is faster than frmv, which uses the full volumetric data. Both achieve times of a few seconds, and it could be less than a second for frmr if the maps are not too large.

Usage (at shell prompt) (same for frmv):
 

./frmr <Situs density map> <PDB structure> -res <number> -cutoff <number> -explor <number>

The input parameters are:

<Situs density map> Low resolution map in Situs format. 

<PDB structure> Atomic structure in PDB format.

-res  Estimated resolution of your density map in Å.  [default -res 15.0]

-cutoff  Density map cutoff value. All density levels below this value are set to zero. [default -cutoff 0.0]

-explor  Controls the number of the best fits to be output. [default -explor 5]

 

Input at program prompt:
None.
 

Output:

The programs generate a PDB file for each peak found. The files are named as follows:

frmr_best*.pdb or frmv_best*.pdb,

where * signifies numbers from 1 to the number of peaks found or the -explor option, whichever is smaller.
frm6 -  FFT-accelerated 6D exhaustive search

Purpose:

This is a rigid-body rotational and translational search tool. It performs the fitting by recasting the problem into a search of 5 angular and 1 linear parameters.

Usage (at shell prompt):
 

./frm6 <Situs density map> <PDB structure> <options>

The input parameters are:

<Situs density map> Low resolution map in Situs format. 

<PDB structure> Atomic structure in PDB format.

The <options> are:

-res <value>  Estimated resolution of your density map in Å.  [default -res 15.0]

-ani <value>   Defines the resolution anisotropy factor (z direction vs. x,y plane)  [default: -ani 1.0]. Allows one to set a different resolution in the z direction vs. the x,y plane. E.g. "-ani 1.5 -res 20" specifies a 30 A resolution in the z direction, and a 20 A resolution in x,y.

-cutoff <value>  Density map cutoff value. All density levels below this value are set to zero. [default -cutoff 0.0]

-corr <number>  This option controls whether the maps are pre-filtered with a Laplacian kernel.
Two values can be given here:

      -corr 0 Standard linear cross-correlation.  The scalar product between the density maps of the low resolution map and the low-pass filtered atomic structure. For resolutions > 10Å this criterion is less discriminative.

      -corr 1 [default] A Laplacian filter is first applied to maximize the fitting contrast. This is the default and is the recommended docking criterion for low resolution docking (up to 25Å resolution). To provide for a more robust algorithm when dealing with cropped experimental data we implemented a mask that filters out hard artificial surfaces. This is done transparently for the user.

-mode <number>  This option controls the scanning of the linear parameter (rho).
Three values can be given for this option: 

      -mode 0 [default]  Automatically select rho scanning mode 1 or 2. This is done according to the relative sizes and shapes of the molecules.

      -mode 1  Determine an initial value of rho and start the search from there.

      -mode 2  Scan the whole range of rho.

-explor <number>  Controls the number of the best fits to be output. [default -explor 5]

Input at program prompt:
None.
 

Output:

This program generates a PDB file for each peak found. The files are named as follows:

frm6_vol_best*.pdb  or  frm6_lap_best*.pdb,

according to whether the -corr option is 0 or 1, and where * signifies numbers from 1 to number of peaks found or the -explor option, whichever is smaller.
